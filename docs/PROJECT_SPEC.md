# 線上聯絡簿正式需求規格

版本：0.1

確認日期：2026-07-18

第一版目標：2026-08-07

## 1. 專案範圍

第一版只供 115 學年度八年六班使用，但所有主要資料均保留班級識別欄位，使未來分享給其他導師時不必重建資料庫。

第一版包含：

1. 學生／家長共用帳號。
2. 教師自行註冊、導師核准及科目授權。
3. 學生名單、科目、數學與英語獨立分組。
4. A 組、B 組及共同作業。
5. 作業長／各科小老師發布作業及第一階段繳交登記。
6. 任課老師管理本科作業及繳交異常。
7. 未完成、未攜帶、遲交、請假、公假、免交及已補交流程。
8. 本週、本月、本學期及本學年統計。
9. 全校公告、班級公告、圖片與已讀紀錄。
10. 向全班公開姓名的榮譽榜。
11. 前往英文單字系統的按鈕。
12. 獨立班級行事曆，學生與教師可切換月份查看，導師可新增、編輯、下架及恢復行程。
13. 管理者 Excel 成績匯入、發布、全班查詢及學生個人成績分析。

第一版不包含：

- 座位表。
- 家長留言。
- 回條照片。
- 作業附件。
- 多班級切換介面。

## 2. 登入與帳號

### 2.1 學生／家長

- 學生與家長共用同一帳號。
- 登入名稱為學生學號。
- 導師建立學生後，由系統產生一次性啟用碼。
- 學生第一次登入時輸入學號與啟用碼，再自行建立密碼。
- 啟用碼使用後立即失效。
- 忘記密碼由導師從後台重設，不依賴學生電子郵件。

### 2.2 任課老師

- 任課老師自行建立帳號及密碼。
- 新帳號狀態為「等待核准」。
- 等待核准期間不能查看班級、學生、作業、公告或成績資料。
- 導師核准後，設定任教班級及任教科目。
- 任課老師只能管理自己被指派科目的資料。

### 2.3 導師

- 導師為本班系統管理者。
- 可管理學生、教師核准、角色、科目、分組、全部作業、繳交紀錄、公告及榮譽榜。
- 可停用帳號及重設學生密碼。

## 3. 角色與權限

| 功能 | 導師 | 任課老師 | 作業長／科目小老師 | 學生／家長 |
| --- | --- | --- | --- | --- |
| 管理學生與啟用碼 | 是 | 否 | 否 | 否 |
| 核准教師及設定科目 | 是 | 否 | 否 | 否 |
| 管理分組 | 是 | 否 | 否 | 否 |
| 查看全科資料 | 是 | 否 | 否 | 否 |
| 查看被指派科目 | 是 | 是 | 是 | 僅適用作業 |
| 發布本科作業 | 是 | 是 | 是 | 否 |
| 第一階段繳交登記 | 是 | 是 | 是 | 否 |
| 修正繳交紀錄 | 是 | 本科 | 否 | 否 |
| 建立公告與榮譽榜 | 是 | 否 | 否 | 否 |
| 查看個人缺交 | 是 | 本科 | 否 | 僅本人 |
| 匯入及發布全班成績 | 是 | 否 | 否 | 否 |
| 查看個人成績 | 是 | 否 | 否 | 僅本人且限已發布 |

小老師登記直接生效，不要求任課老師逐筆確認；任課老師或導師發現錯誤時再修正。所有修正都必須留下操作歷程。

## 4. 科目與分組

- 初始科目包含國文、英語、數學、自然、社會、健體、藝文、綜合及其他。
- 導師可啟用、停用或調整科目順序。
- 數學與英語分組彼此獨立。
- 必須支援數學 A／英語 A、數學 A／英語 B、數學 B／英語 A、數學 B／英語 B。
- 分組紀錄包含學期、生效日與結束日。
- 調整新分組不能改變已發布作業的原始對象。
- 作業發布時建立學生對象快照，確保歷史資料不因後續分組調整而變化。

## 5. 作業

每筆作業包含：

- 班級。
- 學期。
- 科目。
- 作業日期。
- 作業內容。
- 繳交期限。
- 適用對象：共同、A 組或 B 組。
- 發布者與發布時間。
- 啟用或取消狀態。

學生只能讀取作業對象快照中包含自己的作業。共同作業包含全班；分組作業依該科目當時有效分組產生對象。

## 6. 繳交流程

### 6.1 點收

1. 作業發布後顯示「尚未完成點收」。
2. 若全班皆交，點選「全班繳交」完成該次點收。
3. 若有例外，只選取異常學生，不需逐一勾選已繳學生。
4. 小老師可做第一階段登記；任課老師或導師可直接登記或修正。

### 6.2 狀態

- 未完成：計入未繳次數。
- 未攜帶：計入未繳次數。
- 遲交：計入遲交次數。
- 請假：暫時不計入，但必須設定下一次該科上課的追繳期限。
- 公假：暫時不計入，但必須設定下一次該科上課的追繳期限。
- 免交：不計入未繳或遲交次數。
- 已補交：結束提醒，但保留原始原因與歷史。

### 6.3 請假與公假追繳

- 登記請假或公假時必須保存追繳日期。
- 學生於下一次該科上課前完成繳交，不計未繳或遲交。
- 到達追繳日期仍未交，任課老師或導師將紀錄轉為未完成、未攜帶或遲交。
- 轉換後依新狀態計入統計，原始請假或公假原因仍保留在歷程中。

### 6.4 補交與隱藏

- 補交後狀態改為已補交。
- 原始未完成或未攜帶原因仍保留。
- 同一份作業的遲交次數最多計一次。
- 已補交紀錄在待辦提醒中保留 1 天，之後自動從待辦區隱藏。
- 隱藏不是刪除；歷史頁及統計仍可使用該紀錄。

### 6.5 統計

同時顯示：

- 本週：星期一至星期日。
- 本月：日曆月份。
- 本學期：依學期起訖日。
- 本學年：依學年起訖日。

統計以作業繳交期限為歸屬日期；日後若教學流程需要，可改為實際登記日期。

## 7. 公告與榮譽榜

### 7.1 公告

- 類型為全校公告或班級公告，兩者都由導師建立。
- 支援標題、內容、公告圖片、發布日期及到期日。
- 公告圖片存放在 Supabase Storage，資料表只保存路徑與必要資訊。
- 學生帳號點選已讀後，以該學生為單位保存已讀時間。
- 導師可查看已讀及未讀名單。

### 7.2 榮譽榜

- 全班登入者可看到學生姓名、榮譽內容與日期。
- 不在榮譽榜顯示個人成績、缺交次數或其他私密資料。
- 導師可新增、編輯、隱藏及刪除榮譽項目。

### 7.3 班級行事曆

- 學生介面以「聯絡簿／班級行事曆」標籤切換，不將月份月曆塞入聯絡簿首頁。
- 任課老師可從教師工作台切換至唯讀行事曆。
- 支援上個月、下個月、回到今天、跨日行程及點選日期查看完整內容。
- 行程包含標題、分類、日期、全天或時間、地點及詳細說明。
- 分類包含班級活動、學校活動、考試、放假及其他。
- 僅導師可新增與編輯；下架採可復原方式，學生及任課老師只讀取啟用中的行程。

## 8. 個人成績

- 僅管理者可匯入 Excel、查詢全班成績及發布或下架。
- 支援七、八、九年級各三學期、每學期兩次段考，以及四次模擬考。
- 先預覽學生配對與考試內容，確認後才寫入資料庫；匯入後預設不發布。
- 優先用學號配對；來源沒有學號時，以座號與完全相同的姓名配對。
- 同一次考試可重新匯入修正；來源空白不覆蓋舊值，方便日後補入校排。
- 學生只可讀取自己的已發布成績，任課教師目前不開放。
- 空白分數顯示為缺考，且不納入平均與雷達圖計算。
- 學生介面包含單次成績表、七科雷達圖、歷次七科成績及站內規則式學習分析。
- 學習分析不把學生姓名或成績傳送到外部 AI 服務。

## 9. 介面原則

- 手機優先，也能在桌面使用。
- 可愛但不幼稚的 3D 卡片風格。
- 清楚區分共同、A 組及 B 組作業。
- 重要操作使用大按鈕，避免小老師與任課老師誤觸。
- 聯絡簿內提供「前往英文單字系統」按鈕。
- 英文單字系統的實際網址在部署前設定，不寫死於規格文件。

## 10. 非功能需求

- 權限由 Supabase RLS 強制執行，不能只依前端畫面判斷。
- 所有重要寫入保存建立者、修改者及時間。
- 繳交狀態修改保存事件歷程。
- 真實資料匯入前，以測試帳號驗證跨角色資料隔離。
- 第一版完成日期以 2026-08-07 為目標，並採分階段確認。
