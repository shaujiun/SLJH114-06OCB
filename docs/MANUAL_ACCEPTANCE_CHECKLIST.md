# 第一版正式操作驗收表

此表用於正式資料匯入及 GitHub Pages 發布前的最後驗收。測試時不要在文件中記錄密碼、啟用碼或資料庫金鑰。

## 測試環境

- 本機網址：`http://127.0.0.1:4173/`
- 管理員：既有導師管理員帳號
- A 組學生：一位數學 A、英語 A 的測試學生
- B 組學生：一位數學 B、英語 B 的測試學生
- 教師：至少一個待核准帳號及一個已核准帳號
- 手機：至少測試一支直向手機；建議再測試橫向畫面

## 一、管理員桌機驗收

- [ ] 登入後，後台總覽四張摘要卡在同一橫行。
- [ ] 「目前學年度」欄較寬，「啟用科目」欄較窄，文字未被截斷。
- [ ] 可建立共同、A 組及 B 組作業。
- [ ] 已發布作業固定依「共同 → A 組 → B 組」排列。
- [ ] 同一組內依期限由新到舊排列。
- [ ] 作業右側只顯示月日、時間、發布者及人數，不顯示年份與秒數。
- [ ] 「全班已繳交」與「登記例外學生」可正常操作。
- [ ] 可建立、編輯、下架及恢復行事曆活動。
- [ ] Excel 行事曆可先預覽，再匯入勾選項目；重複資料會略過。
- [ ] 公告圖片、已讀名單及多人榮譽榜可正常操作。

## 二、學生聯絡簿驗收

### A 組測試學生

- [ ] 只能看到共同與 A 組作業，看不到 B 組作業。
- [ ] 上方標籤依序為「聯絡簿、公告欄、班級行事曆」。
- [ ] 聯絡簿頁面不顯示公告與榮譽榜。
- [ ] 公告欄頁面同時顯示公告與榮譽榜。
- [ ] 作業不顯示發布者，期限只顯示月日。

### B 組測試學生

- [ ] 只能看到共同與 B 組作業，看不到 A 組作業。
- [ ] 英語 B 組與數學 B 組作業合併在同一個「B 組」區塊。
- [ ] 不同科目的共同作業合併在同一個「共同」區塊。
- [ ] 公告「我已閱讀」可成功儲存，重新整理後仍顯示已閱讀。
- [ ] 行事曆只能查看，沒有新增、編輯、下架按鈕。

## 三、教師與小老師權限驗收

- [ ] 未核准教師登入後看不到班級、學生、作業及公告資料。
- [ ] 已核准教師只能發布被授權科目的作業。
- [ ] 同一位教師獲授權多科後，可切換並發布各科作業。
- [ ] 數學或英語教師可依授權操作 A、B 組。
- [ ] 教師可使用「全班已繳交」，並只登記例外學生。
- [ ] 教師可查看行事曆，但不能修改行事曆。
- [ ] 科目小老師只能操作被指派的科目與組別。
- [ ] 作業長可操作允許的全部科目，但不能進入管理員設定。
- [ ] 小老師不能修改教師已建立的例外紀錄。

## 四、手機直向驗收

- [ ] 可完成登入、登出及重新整理。
- [ ] 三個學生功能標籤均可點擊，沒有超出版面。
- [ ] 作業列的科目、內容、期限及狀態沒有互相重疊。
- [ ] 公告圖片可完整查看，不會撐破卡片。
- [ ] 行事曆可切換月份、點選日期並查看當日活動。
- [ ] 教師可完成全班繳交與例外學生登記。
- [ ] 表格較寬時可在區塊內左右捲動，不會使整頁橫向溢出。
