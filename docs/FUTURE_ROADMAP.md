# 未來功能規劃

本文件同時記錄已落地的擴充方向與仍待後續製作的功能，實際完成狀態以各節標示為準。

## 一、各科學習系統入口

### 目標

- 聯絡簿保留「前往各科學習系統」入口。
- 進入後先顯示科目選擇頁，再前往各科的學習內容。
- 目前先提供英語單字系統，未來可逐步加入其他科目。
- 學生使用同一組帳號，不必為每一個學習系統重新建立帳號。

### 共用方式

- 英語單字系統與線上聯絡簿目前共用同一個 Supabase 專案及帳號系統。
- 各網站可分別由 GitHub Pages 發布，再由各科學習系統入口集中串接。
- 不在網址中傳遞密碼或私人資料；登入狀態及權限仍由 Supabase Auth 與 RLS 管理。
- 若未來網站改用不同網域，需重新確認跨網站共用登入的方式。

### 未來可加入的教材形式

- 各科線上桌遊或教學遊戲。
- 各科學習重點整理。
- 心智圖或概念關係圖。
- 學科漫畫或情境式學習內容。
- 其他適合手機與電腦使用的互動教材。

### 建議製作順序

1. 完成各科選擇首頁，先接上現有英語單字系統。
2. 建立統一的科目名稱、圖示、網址、教材類型及啟用狀態設定。
3. 視需求增加管理員新增、排序、隱藏學習系統連結的功能。
4. 新教材各自製作完成後，再加入科目入口，不影響既有聯絡簿。

## 二、學生個人成績系統（第一階段已完成）

### 介面位置

- 學生介面依序為「聯絡簿、公告欄、班級行事曆、個人成績」。
- 個人成績使用獨立頁面，手機版標籤可橫向捲動並維持單行文字。

### 成績內容

- 已支援 18 次段考及 4 次模擬考目錄。
- 管理者可先預覽 Excel 配對結果，再確認匯入、查詢全班及個別發布。
- 學生可查看已發布的單次成績、七科雷達圖、歷次七科表格與平均。
- 已加入不連外的站內規則式分析、讀書方法與正向鼓勵。

### 成績分析方向

- 後續可加入各科折線圖、段考與模擬考交叉比較及更細緻的長期趨勢。
- 若未來啟用真正的生成式 AI，需另行確認家長告知、個資去識別、資料保存及費用。

### 權限與隱私

- 學生及家長只能查看自己的成績。
- 導師可查看、匯入及管理全班成績。
- 任課老師目前不開放成績權限，日後若需開放再新增本科政策。
- 成績資料必須由 Supabase RLS 限制，不能只靠前端隱藏按鈕。
- 不在榮譽榜、公告或其他學生可見區塊顯示個人成績。
- 所有新增及修改需保存操作者與時間。

### 資料模型

- 正式成績模組使用 `grade_exam_periods`、`grade_import_batches` 與 `student_grade_results`。
- 舊版預留的 `assessment_periods`、`assessments` 與 `student_scores` 暫時保留，但目前介面不使用。
- 重新匯入同一次考試會更新既有紀錄，並保留先前已有、這次來源為空白的欄位。

## 三、目前仍不執行的事項

- 不開放任課老師查看或匯入成績。
- 不把學生資料送到外部生成式 AI。
- 不自動匯入尚未完成學生帳號配對的真實成績。
